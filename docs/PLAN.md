## 背景・概要

今回は、「文化祭」で用いるアプリを作成しようとしています。

文化祭では、立場ごとに幾つかの要望や問題があります。

- 文化祭運営
  - 各ブースの混雑を解消し、参加者の満足度を向上させたい
- 出店者（主にクラス）
  - 自信のブースにたくさん人が来てほしい
- 参加者（生徒や外部の人）
  - どこで何がやっているのか、どのくらい混んでいるのかがわからない

このような要望・問題点を解消するために、「文化祭デジタルスタンプラリーアプリ」を開発します。
このアプリでは、文化祭運営がブースごとにブース情報、スタンプ画像、場所を登録する。そうすると、QRコードが1ブースごとに発行される。
そのQRコードを出店者は店内に掲示することでスタンプラリーをそのブースで行うことが出来る。
参加者は、ブースに行ってQRコードを読み込むことでスタンプラリーを実施できる。

スタンプラリーを集めることで景品などと交換できることで、多くのブースを周るきっかけにもなり、楽しむコンテンツも増える（今回は景品等は検討しない）

また、QRコードの読み込み量を元に、そのブースの現在の混雑状況を見ることが出来る。
これによって、アプリ上に混雑状況を表示して、現在空いているブースを確認したり、行きたいブースがどのくらい混んでいるのかを知ることが出来るため、移動の計画を立てることに役立つ.

## バックエンドAPI

- users
  - POST
    - 新たな匿名ユーザーを作成
- users/me/stamps
  - GET
    - 現在のユーザーが持つスタンプを取得
- scans
  - POST
    - QRコードをスキャンしたログを登録。読み込んだQRコードに対応するスタンプ情報・画像を返す
- booths
  - GET
    - ブースの情報を取得
- booths/{booth_id}
  - POST
    - ブース情報を登録。QR生成に必要な情報を返す
